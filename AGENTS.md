# 智印联创开发前检查

每个开发任务开始前必须执行以下步骤：

1. 阅读 `docs/development-capability-matrix.md`，按任务类型选择最小必要 Skill 或插件。
2. 阅读被选 Skill 的完整 `SKILL.md`，并在进度更新中说明它如何影响本任务。
3. 新功能和行为变更遵循测试驱动：先写失败测试，再实现，再运行完整相关测试。
4. 联网采集只使用白名单公开来源，保留来源、许可、采集时间与审核状态；采集结果不得自动公开。
5. 任何页面不得展示无法核验的合作、订单、价格、预测、节省额或经营成绩。
6. 演示数据只能存在于评审演示状态，不得写入真实订单、报价、集采或运营统计。
7. 完成前运行类型检查、Lint、生产构建、相关单元测试和核心浏览器流程。

设计基线：工业编辑式，`DESIGN_VARIANCE=6`、`MOTION_INTENSITY=6`、`VISUAL_DENSITY=5`。主色为结构海军蓝，唯一操作强调色为印刷橙。所有动效必须说明其信息作用，并支持减少动态偏好。
