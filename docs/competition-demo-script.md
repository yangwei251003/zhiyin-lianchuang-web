# 智印联创比赛演示脚本

建议时长：3–5 分钟。演示使用独立评审入口，演示数据不会写入真实订单、报价、集采或运营统计。

## 0:00–0:25 先讲一句核心

> 智印联创不是单独展示一个订单页面，而是把需求方、印刷厂和原料供应商放进同一条可追踪的印刷产业协同闭环：需求发布、产能与报价响应、原料供货方案，最后回到可核验的采购决策。

## 0:25–0:45 打开 `/review`

输入评审码，说明这是独立演示入口。不要先从首页菜单慢慢找功能，评委需要先看到完整故事。

## 0:45–2:20 打开 `/review/workspace`

按右侧角色切换，严格按这个顺序讲：

1. **需求方**：点击“发布演示需求”。讲校园开放日宣传物料如何从一个真实需求开始，系统记录规格、数量和交付目标。
2. **印刷厂**：切换角色，点击“提交演示报价”。强调印刷厂看到的是同一需求，不需要线下反复转述。
3. **原料供应商**：切换角色，点击“提交演示供货方案”。强调原料供应商补上了纸张、起订量与交期这一环。
4. 看三个角色都出现“已完成”。这时停下来指出：同一个项目在三种身份之间传递，协同闭环完成，而且演示状态与真实业务表隔离。

## 2:20–3:00 打开 `/prediction/白卡纸`

只讲三点：

- 页面显示纸种、规格、地区、单位、更新时间和来源；
- 有效报价才展示，过期或缺少来源时明确显示数据不足；
- PPI 和纸浆期货只作为宏观参考，不冒充白卡纸等成品纸报价。

这部分的亮点是“可信边界”，不要把“预测”说成已经拥有全市场实时数据。

## 3:00–3:40 回到 `/orders` 或 `/purchase`

快速展示真实业务入口：公开需求、采购意向和供应方案。只说明真实用户登录后可以继续完成业务流程，不现场填写真实主体资料。

## 3:40–4:00 收束

> 我们解决的不是把页面做得更像交易网站，而是把印刷产业链中最容易断开的三段信息接起来：需求、生产响应和原料供给；同时对数据来源、演示隔离和权限边界负责。

## 评委可能追问

**为什么不直接展示每种纸的实时预测？**

因为公开、免费且允许商业使用的中国成品纸实时 API 并不存在稳定覆盖。平台宁可显示数据不足，也不把纸浆、PPI 或未经授权的网络数字伪装成具体纸种报价。拿到供应商授权 API 或持续报价后，可以沿现有白名单采集、审核、发布流程接入。

**演示会不会污染真实业务？**

不会。评审会话使用独立签名 HttpOnly 会话，角色进度保存在前端演示状态，不调用真实业务写入接口。

**项目的核心竞争力是什么？**

三角色协同闭环、可追溯的数据治理，以及把产教实践放进真实产业问题，而不是只做一个单点报价或展示页面。
