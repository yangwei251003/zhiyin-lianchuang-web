# 智印联创工业编辑式视觉设计

## 目标

在不改变现有功能、数据真实性约束、技术栈或移动端可用性的前提下，为智印联创建立更成熟、克制、可信的印刷产业服务平台视觉体验。

## 设计方向

采用“印刷样张式工业编辑设计”。参考 MOO 对实体印刷材料的强调、VistaPrint 对任务入口的清晰组织，以及 Linear 的信息层级和留白纪律；不复制其页面、文案或品牌元素。

页面以真正的白色为基础色，海军蓝作为信息与结构色，橙色只用于主要行动。取消大面积渐变、玻璃拟态、霓虹光效、悬浮数字和未经证实的 AI 视觉符号。品牌 Logo 是首屏信号，印刷相关的裁切线、套准标记和网点纹理仅以细节方式出现，不成为装饰主体。

## 设计系统

- 背景：`#FFFFFF` 主背景，`#F6F7F8` 次级工作区；禁止暖米色背景。
- 文字：`#172033` 标题，`#425066` 正文，`#6B7280` 辅助信息。
- 品牌：`#173B63` 结构色，`#D97706` 主操作色，`#047857` 已验证或正常状态。
- 边框：`#D9DEE6`，以 1px 线和层次留白取代大阴影。
- 圆角：业务面板最多 8px；按钮、筛选和标签采用 4px 或直角。
- 排版：标题使用现有中文无衬线字体栈，正文维持 16px 与舒适行距；避免超大营销式标题和负字距。
- 图标：仅使用已有 lucide-react 描边图标，控制在 16-20px，作为功能提示而非装饰。

## 页面与交互

### 全局框架

Header 保持单行清晰导航：品牌、核心业务入口、一个主要操作、移动端菜单。Footer 采用信息列与规则说明，减少不必要的徽章式元素。聚焦态、悬停态和禁用态均使用颜色、边框与文本共同表达，不能只依赖动画。

### 首页

首屏采用左侧清晰价值主张与右侧“服务入口索引”的开放布局。右侧不使用伪仪表盘、增长数字或 AI 插画；使用纸张裁切线和业务清单表达印刷属性。核心服务采用不等宽的编辑式分段，而不是重复卡片瀑布流。真实调研数据和数据授权状态保持来源说明。

### 订单、集采与纸价

三个业务页统一为“页面标题 + 主要操作 + 筛选工具条 + 内容列表/空状态”的工作台节奏。筛选项使用带标签的控件；空状态显示下一步可执行操作，但不承诺平台能力或交易结果。纸价页保留来源、规格、区域、单位、更新时间和原始链接，历史不足时只显示诚实的数据不足状态。

### 表单、反馈与个人中心

输入控件、步骤说明与错误提示使用高对比边框和明确标签。提交按钮仅在满足前置条件时启用。反馈页面保留数据保存和隐私提醒，不展示虚假成功或处理时效。

### 移动端

在 390px 宽度下，页头仅显示品牌和菜单；核心行动全宽或自适应换行。筛选控件垂直排列，列表信息不截断。底部导航保留四个最高频入口；浮层和抽屉不遮挡表单提交操作。

## 动效与媒体

动效仅用于页面切换、列表进入、抽屉和交互反馈，时间控制在 160-240ms。避免连续循环装饰、视差光效和无意义的数值滚动。现有不符合方向的未来感印刷插画不作为首页主视觉；在没有已授权真实生产图片时，使用代码原生的印刷标记和品牌 Logo，而不引入来源不明的网络图片。

## 验收

1. 视觉系统在首页、订单、集采、纸价、反馈和认证页面中一致。
2. 任何页面不显示虚构价格、交易量、增长率、节省额或 AI 预测结论。
3. 桌面与 390px 移动端不存在横向溢出、重叠、裁切或无法操作的控件。
4. `pnpm exec tsc --noEmit`、`pnpm run lint` 和 `pnpm run build -- --webpack` 通过；浏览器检查核心流程与空状态。
5. GitHub 推送和 Vercel 生产部署仅在验证完成后执行，敏感环境变量不进入版本库。
